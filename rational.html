Rational

GitHub Links

In the Rexburgians home page I changed all of the GitHub links so that they go directly to the GitHub page and not the GitHub tab.
The reason I did this is because most people would prefer to click their mouse as few times as possible. They want to get there quickly, and that's why I reduced the amount of times they have to click to get over to GitHub.

Spaced HTML Tags

Several tags were not spaced together, or some tags would start at the same line other tags ended. I respaced the tags properly so that the code is more easy to read.
"Code needs to be legible, follows standard coding guidelines, and commented properly so that others may have an easier time reading your code."

Reorganized Style Tags

I removed all inline style tags because I think they are clunky and disorganized. I moved several in-document styles to styles.css because I thought they were styles that should be shared with other documents. The only styles that were left alone were styles that I thought should belong to that page only and no other.
"Styles should be consistent throughout all of the webpages on your website. They should have a consistent look and theme."

Images Jitter a Lot Less in App

I thought that the images moved around too much and it was more distracting than stylish. I made them move a whole lot less, but hope to improve on those transitions through javascript or CSS3.
"Images should be in the background and not distract the user from the content of the page."

Changed Email Address in Contact Tab

I thought hen14020@byui.edu would look more professional than jasonhhotstuff@aol.com
"Your web page should target the appropriate audience and do as little as it can to offend the user in any way."

More Content in Test App Tab

I added some more information regarding the Mobile App version of Rexburgians
There is no guideline standard reason why I did this, I just wanted to give more relevant information to the user.
